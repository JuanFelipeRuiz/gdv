## GDV Repository

This repository contains the data, notebooks and final
presentation and report artefacts for the GDV project on Swiss
passenger-car fuel types. It focuses on creating visualizations to answer the following three questions:


1. How did the fuel type composition of cars in Switzerland evolve over time?
2. How do fuel types differ across Swiss cantons?
3. How has the adoption of electric vehicles evolved in different cantons?

## Project Structure

- `data/`
  - `registered.csv`: FSO vehicle-stock data for passenger cars by canton, holder
    type, fuel type, year, and observed count.

- `src/`
  - `preparation.py`: shared data preparation pipeline. It loads *data/registered.csv*, filters passenger cars, removes total rows and summarizes fuel types into the five categories used for plotting.
  - `shared_configs.json`: shared plotting configuration, including output path,
    fuel category order, colour palette, text styles, and save settings.
  - `a_fuel_over_time.ipynb`: creates the fuel-type-over-time area charts.
  - `b_fuel_canton.ipynb`: creates canton-level fuel composition charts, including map and stacked bar variants.
  - `c_canton_comparison.ipynb`: creates the Aargau BEV comparison charts, including map, bar, and lollipop variants.
  - `eval_tool.py`: small local evaluation tool used for pairwise plot
    comparisons and participant notes. Evaluation results can only be found inside the final report. The application was vibe coded with the help of Claude.

- `docs/`
  - `graphic_own/`: exported plot PNGs generated by the notebooks. Needed for eval_tool.py and the final report.

  - `main.pdf`: final written report.
  - `presentation.pdf`: final presentation slides.


## Reproducing the Data

The data was downloaded manually by navigating to the FSO's STAT-TAB database and downloading the "registered vehicles by canton, holder type, fuel type, year" dataset as a CSV file. This repo does not contain the steps for the selection of filters and download. Therefore it is limited in reproducing the data only given the existing `data/registered.csv` file or one that follows the same structure. The link to the
API user interface is inside the final report.



## Reproducing the Plots
First we need to prepare our Python environment. We recommend using a virtual environment and installing the dependencies from `requirements.txt`:

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Afterwards we can run the notebooks from the `src/` directory so the relative paths in
`preparation.py` and `shared_configs.json` resolve correctly. It may happen that the coding IDE asks to install some additional package for the jupyter notebook support, please allow it to do so.

The suggested order to run the notebooks is as follows, but they can also be run independently:

1. `src/a_fuel_over_time.ipynb`
2. `src/b_fuel_canton.ipynb`
3. `src/c_canton_comparison.ipynb`


The notebooks themselves are focused on plotting and use the prepared data from `preparation.py`. For understanding the data preparation steps, see `src/preparation.py`. It also uses the shared plotting configuration from `src/shared_configs.json` for consistent styling and saving settings across all plots. Each plot is saved to `docs/graphic_own/` as a PNG file, which is then used for the final report and evaluation with the following naming convention:

- `vis_a_*`: fuel type development over time.
- `vis_b_*`: fuel type shares across cantons.
- `vis_c_*`: BEV comparison between Aargau and other cantons.

## Reproducing the Evaluation

The evaluation was conducted using a local Python application defined in `src/eval_tool.py`.
To start a local instance of the evaluation tool, run the following command from the project root:

```bash
python -m src.eval_tool
```

It should open two windows. The one with the button on the right is for the person leading the evaluation, the other one without the red button is for the participant. If available, move one of the windows to a second screen or just resize them so both windows are visible at the same time.
